# Devin — 能力说明书

> 本文档描述 Devin 的能力边界、触发方式和工作流程。
> 读者：人类开发者、其他 AI Agent、系统集成方。

---

## 身份

Devin 是一个自主工作的 AI 软件工程师。它通过 GitHub Issue 和 PR 接收需求，独立完成从需求分析到代码部署的完整开发流程。

它不是一个对话机器人——它是一个**执行者**。给它一个任务，它会交付代码。

---

## 触发方式

### 何时触发

在 GitHub Issue 或 PR 的评论中包含 `@devin`，Devin 即被激活。

| 场景 | 示例 | 触发 |
|------|------|------|
| 新建 Issue 带 @devin | Issue body 含 `@devin 帮我做一个登录页` | ✅ |
| 在已有 Issue 评论 @devin | `@devin 请开始实现` | ✅ |
| 在 PR 评论 @devin | `@devin 按 review 意见修改` | ✅ |
| Issue 不含 @devin | 普通 Issue | ❌ 不触发 |
| Devin 自己的评论 | 以 🤖 / 📋 / ✅ 开头的评论 | ❌ 自动过滤 |

### 触发链路

```
GitHub Webhook (issue_comment / issues)
  → Devin Server (验证签名 + 检测 @devin)
  → Agent Loop (LLM 自主决策 + 执行动作)
  → GitHub (评论 / 提交代码 / PR / merge)
```

## 能力清单

### 可以做的事

| 能力 | 说明 | 自主程度 |

| **创建 PR** | 创建 Pull Request，含方案摘要和变更列表 | 全自主 |

| **代码校验** | 提交前自动检查括号匹配、JSON 合法性、空文件、markdown 残留 | 全自主 |

| **智能上下文** | 自动读取 package.json、tsconfig 等理解项目技术栈 | 全自主 |

### 不能做的事（当前版本）

| 限制 | 原因 | 计划 |
|------|------|------|
| 运行测试 | 当前通过 API 生成代码，不在本地执行 | 接入 Claude Code Agent SDK 后支持 |
| 运行构建 | 同上 | 同上 |
| 访问私有 npm/pip 包 | 无本地运行环境 | 需要容器化执行环境 |
| 处理图片/设计稿 | LLM 输入限于文本 | 接入多模态模型 |
| 跨仓库操作 | 当前每次只操作触发的仓库 | 架构已预留 |
| 并发处理 | 多个 Issue 串行处理 | 加任务队列 |
| 数据库操作 | 无数据库访问权限 | 按需配置 |

## 工作流程

### 典型流程（由 Devin 自主决定）

```
1. 收到 @devin → 分析需求
   ├── 需求不清晰 → 在 Issue 提问 → 等人类回复 → 重新开始
   └── 需求明确 ↓

2. 评论方案摘要

3. 创建分支 → 生成代码 → 提交文件

4. 创建 PR

5. 自我审查
   ├── 发现问题 → 修复代码 → 重新提交
   └── 审查通过 ↓

6. 合并 PR → 部署（如适用）→ 关闭 Issue

7. 检查 README.md 运维信息是否完整
   ├── 缺失但能推断 → 自动补充（部署方式、项目结构等）
   └── 缺失且无法推断 → 在 Issue 向需求提出者提问

8. 汇报结果
```

### 关键特性：自主决策

Devin 的工作流程不是硬编码的。每一步由 LLM 根据当前上下文自主判断：

- **简单任务**（如创建一个静态页面）：可能 5 轮迭代内完成全部流程
- **复杂任务**：可能需要多轮，包含反复的 review → 修复循环
- **不确定的任务**：会先提问，等人类澄清后再继续
- **静态网页类任务**：会自动部署到 GitHub Pages

每轮迭代中 Devin 可以批量执行多个动作（如同时创建 PR + 评论 + merge）。

## 如何与 Devin 协作

### 对人类

```markdown
<!-- 提需求 -->
@devin 帮我实现一个用户登录页面，用邮箱+密码，支持记住我功能

<!-- 补充说明 -->
@devin 登录成功后跳转到 /dashboard

<!-- 要求修改 -->
@devin 按钮颜色改成蓝色，输入框加 placeholder

<!-- 让它自己全部搞定 -->
@devin 做完后自己 review 和 merge
```

### 对其他 Agent

如果你是另一个 AI Agent，需要调度 Devin 完成开发任务：

1. 通过 GitHub API 创建 Issue，body 中包含 `@devin`
2. Issue body 中清晰描述：
   - 要做什么（功能描述）
   - 文件放在哪里（路径）
   - 技术约束（语言、框架）
   - 完成标准（什么算做完）
3. Devin 会自动工作并在 Issue 中汇报结果
4. 你可以通过 GitHub API 轮询 Issue 状态和评论来获取进度
5. 如果需要 Devin 补充修改，在 Issue 评论中 `@devin` + 新指令

### 需求描述的最佳实践

| 好的需求 | 不好的需求 |
|---------|-----------|
| `创建 games/tetris/index.html，用纯 HTML/CSS/JS 实现俄罗斯方块` | `做个游戏` |
| `在 src/utils/date.ts 中添加一个 formatDate 函数，输入 Date 返回 YYYY-MM-DD` | `加个工具函数` |
| `修复 login 页面点击提交后没有 loading 状态的 bug，文件在 src/app/login/page.tsx` | `有个 bug` |

关键原则：**说清楚做什么、放在哪、用什么技术**。
